# 📱 WebApp Store Launcher

Современный, быстрый и автономный каталог-лаунчер для Прогрессивных веб-приложений (PWA), написанный на чистом **PHP** и **SQLite**. 

Проект спроектирован как полноценная PWA-витрина с акцентом на мобильный тач-интерфейс, автоматическую темную тему, поисковую оптимизацию (SEO) и высокий уровень доверия пользователей.

---

## 🌟 Ключевые особенности (UVP)

*   **⚡ Мой рабочий стол (Лаунчер):** Главный экран использует `localStorage`. Посещенные пользователем приложения мгновенно выводятся сверху в виде круглых нативных ярлыков для быстрого перезапуска.
*   **🛡️ Система доверия (РФ Реестр и Верификация):** 
    *   Индикатор **`РФ 🛡️`** — парсер автоматически проверяет домены через DNS и реестры безопасности. Легальные сайты в зоне `.RU`/`.РФ` получают этот маркер защиты.
    *   Индикатор **`✓ Проверено`** — маркер ручной модерации человеком для защиты от спама.
*   **📦 100% Автономность контента:** Парсер физически скачивает манифесты, лучшие по качеству иконки и скриншоты в локальные папки. Витрина не сломается, если сторонний сайт упадет.
*   **🔒 Закрытый REST API:** Наполнение каталога скрыто от обычных пользователей. Добавление происходит через безопасный POST-запрос с секретным Bearer-токеном.
*   **📱 Нативный мобильный UX:** Полный отказ от веб-эффектов (запрет выделения, скрытые скроллбары, активный отклик кнопок `:active` вместо залипающего `:hover`, поддержка Safe Areas для «челок» смартфонов).
*   **🌙 Автоматический Dark Mode:** Тема адаптируется под системные настройки смартфона на чистом CSS (без мигания экрана при загрузке страницы).
*   **🔍 SEO-Ready:** Рендеринг на стороне сервера (SSR). Внедрена микроразметка `SoftwareApplication` (для желтых звезд в выдаче Google/Яндекс), `BreadcrumbList` и `FAQPage`. Имеется динамический `sitemap.php`.

## 🗂️ Структура файлов проекта

```text
/webapp-store/
│
├── config.php            # Ключевые настройки: секретный токен доступа и путь к БД
├── database.db           # База данных SQLite (создается автоматически)
├── manifest.json         # PWA-манифест самого каталога
├── sw.js                 # Сервис-воркер магазина для включения standalone-режима
│
├── index.php             # Главная страница: поиск, чипсы категорий, рабочий стол
├── app.php               # Карточка PWA: скриншоты, смарт-инструкция, похожие приложения
├── parser.php            # Ядро: скачивание манифеста, автокатегории, выкачивание картинок
├── api_add.php           # Закрытый REST API эндпоинт для удаленного добавления PWA
│
├── robots.txt            # Инструкции для поисковых роботов
├── sitemap.php           # Динамическая XML-карта сайта, обновляемая автоматически
│
└── uploads/              # Локальное хранилище медиафайлов
    ├── icons/            # Папка для скачанных иконок приложений
    └── screenshots/      # Папка для скачанных скриншотов галереи
```

## 🚀 Быстрый старт и развертывание

### 1. Требования
*   PHP 7.4 или выше.
*   Установленные PHP-расширения: `pdo_sqlite`, `curl`, `mbstring`.

### 2. Установка
1. Клонируйте репозиторий или перенесите файлы в рабочую директорию вашего сервера.
2. Создайте папку `uploads` и подпапки для медиафайлов:
   ```bash
   mkdir -p uploads/icons uploads/screenshots
   ```
3. Выдайте права на запись для веб-сервера (актуально для Linux/Ubuntu):
   ```bash
   chmod -R 775 uploads
   chown -R www-data:www-data uploads
   ```

### 3. Настройка
Откройте файл `config.php` и укажите ваш секретный токен авторизации для наполнения базы:
```php
<?php
// config.php
define('API_SECRET_TOKEN', 'ВАШ_СЛОЖНЫЙ_СЕКРЕТНЫЙ_ТОКЕН_ЗДЕСЬ');
define('DB_PATH', 'sqlite:database.db');
```

### 4. Локальный запуск
Для тестирования на локальном компьютере запустите встроенный сервер PHP внутри папки проекта:
```bash
php -S localhost:8000
```
После этого откройте в браузере: `http://localhost:8000`.

## 🤖 Наполнение каталога через API

Чтобы добавить новое PWA-приложение в ваш магазин, отправьте `POST` запрос на эндпоинт `api_add.php` с использованием вашего токена. В теле запроса передайте JSON с параметром `"url"`.

### Пример запроса через cURL (в терминале):
```bash
curl -X POST http://localhost:8000/api_add.php \
  -H "Authorization: Bearer ВАШ_СЛОЖНЫЙ_СЕКРЕТНЫЙ_ТОКЕН_ЗДЕСЬ" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://telegram.org"}'
```

### Пример успешного ответа (JSON):
```json
{
  "status": "success",
  "message": "Приложение успешно обработано. Манифест прочитан, иконки и скриншоты сохранены локально."
}
```

## 🛠️ Управление модерацией (Для администратора)

Так как проект сохраняет максимальную легкость, в нем нет тяжелой панели администратора. Управлять флагами ручной проверки и белого списка на стадии MVP удобнее всего напрямую через базу данных `database.db` с помощью любой бесплатной программы (например, **DB Browser for SQLite**):

*   `is_verified` (0 или 1) — переключите в `1`, чтобы на карточке приложения зажглась зеленая галочка **`✓`**.
*   `is_whitelisted` (0 или 1) — выставляется автоматически парсером, но вы можете вручную переписать значение, чтобы управлять синим бейджем **`РФ 🛡️`**.

## 🔒 Безопасность и SEO на проде

1.  **Важное требование HTTPS:** Технология PWA работает строго по защищенному протоколу HTTPS. После переноса на реальный хостинг обязательно подключите бесплатный SSL-сертификат (например, Let's Encrypt).
2.  **Индексация:** Не забудьте изменить домен по умолчанию в файлах `robots.txt` и `app.php` на ваш реальный рабочий адрес, чтобы поисковые роботы Яндекса и Google корректно читали `sitemap.php` и строили сниппеты со звездами рейтинга.
